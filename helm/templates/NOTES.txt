PREREQUISITES — VERIFY THESE BEFORE FILING BUGS
=============================================================================

1. REQUIRED SECRETS
   All 7 secrets must exist before pods can start:

   kubectl get secrets -n {{ .Release.Namespace }}

   Expected secrets:
     ecr-registry-secret
     traefik-tls
     {{ .Release.Name }}-rabbitmq-secret
     {{ .Release.Name }}-mongo-init-secret
     {{ .Release.Name }}-opensearch-admin-secret
     {{ .Release.Name }}-hybridsearch-oirag-secret
     {{ .Release.Name }}-hybridsearch-remoteagent-secret

   Missing secrets? Re-run: kubectl-create-secrets.ps1 or aks-create-secrets.ps1

2. OPENSEARCH: vm.max_map_count REQUIREMENT
   OpenSearch requires vm.max_map_count >= 262144 on every cluster node.
   Without this, OpenSearch will fail to start with an error about bootstrap checks.

   Check: kubectl exec -n {{ .Release.Namespace }} <opensearch-pod> -- sysctl vm.max_map_count
   Fix:   sysctl -w vm.max_map_count=262144  (run on the node, or via DaemonSet)

3. OPENSEARCH ADMIN PASSWORD (version 2.4.1 note)
   The OPENSEARCH_INITIAL_ADMIN_PASSWORD env var is IGNORED by OpenSearch 2.4.1.
   The default demo password is in effect until you either:
   a) Upgrade OpenSearch to >= 2.12 (recommended), OR
   b) Run securityadmin.sh inside the pod with a custom internal_users.yml

   The OpenSearch admin username is always: admin
   The password you intend to use is in the secret:
     kubectl get secret {{ .Release.Name }}-opensearch-admin-secret \
       -n {{ .Release.Namespace }} \
       -o jsonpath='{.data.OPENSEARCH_INITIAL_ADMIN_PASSWORD}' | base64 -d

4. ECR TOKEN EXPIRY
   AWS ECR tokens are valid for 12 hours. If pods show ImagePullBackOff,
   re-run the secrets setup script to refresh the ECR credentials.
